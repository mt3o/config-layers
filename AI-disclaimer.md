# Is this library AI generated? Vibe coded? 

No. 

# Why it was created? How it happened?

This library was born after a few years of experience with my work setup where we rely on very similar in-house tool. However the tool we use - lacks some features, and 
is affected by some sub-optimal architectural choices I took long time ago. So - this is how i should have created back then, with knowledge i gathered over the last few years. 

The interfaces this lib has - are created by me, by hand. Main unit tests are created by hand. The implementation is done by hand as well. 

AI helped to generate more tests and examples, as well as assisted in playing with typescript type system. AI generated unit tests where i did not established enough coverage. Every line of code the AI created was - carefully checked by me. 
The library was peer-reviewed, had some fixes done, all by hand. Some JSDOC was improved with help of AI. The tool I use - is Junie from Jetbrains, with the default LLM model. 
Commit descriptions are created by AI. Yes, im that developer who hates creating meaningful commit messages ;p 

# How the AI was used?

- To brainstorm the idea and look for design flaws. 
- To make nice documentation. 
- To add more unit tests and examples. 
- To write commit messages.
